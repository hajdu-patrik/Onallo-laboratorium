# Dashboard UI/UX Design Process (EN)

This document briefly summarizes how the AutoService dashboard UI/UX plan was produced with AI support, MCP orchestration, and the Pencil design environment.

## Quick Summary

The objective was to create a clean, usable dashboard that can be implemented on the frontend side with low friction. The focus was not only visual quality, but also structural consistency.

AI accelerated design iteration, while MCP provided precise node-level control. This enabled multiple refinement cycles in a short timeframe.

---

## Methodology

### 1) Prompt-based design

Natural-language specifications were used to define:

- which screens we need,
- where key elements should be,
- what color and theme rules to follow,
- which language variants to build.

### 2) MCP-based orchestration

MCP connected design intent with the design file. This enabled:

- read the current state,
- apply batch updates,
- edit specific nodes precisely,
- reuse the same layout logic across screens.

### 3) Implementation in Pencil

All final variants were built and refined in Pencil:

- Login,
- 404,
- Dashboard,
- calendar layout,
- EN/HU and Light/Dark versions.

---

## Closing

In this project, AI + MCP + Pencil worked as a reliable combination: fast, precise, and consistent. This workflow is especially effective when strong UI quality is required but designer intuition/knowledge is lacking.
